\chapter{Limites et Contraintes}
\label{chap:limitations}

\section{Limites Techniques}

\subsection{Dépendance à Ollama}

\begin{itemize}
    \item \textbf{GPU} : Performances dégradées sans GPU
    \item \textbf{Démarrage} : 30-60 secondes pour charger modèles
    \item \textbf{VRAM} : 4-8 GB minimum pour VLM
    \item \textbf{Latence} : 2-5 secondes par requête IA
\end{itemize}

\begin{warningbox}[Impact]
Sans GPU, temps de réponse 10-30 secondes ou timeout sur requêtes complexes.
\end{warningbox}

\section{Limites Fonctionnelles}

\begin{itemize}
    \item Modèle RBAC simple avec 4 rôles fixes
    \item Pas de permissions granulaires par ressource
    \item Pas de multi-tenancy natif
    \item Workflow intervention linéaire (pas de branches)
    \item Pas de notifications push temps réel
    \item Pas de générateur de rapports personnalisés
\end{itemize}

\section{Limites IA}

\begin{itemize}
    \item \textbf{Données requises} : Min. 20-30 interventions par équipement
    \item \textbf{Précision RUL} : MAE typique 15-30 jours
    \item \textbf{Hallucinations} : Réponses parfois incorrectes
    \item \textbf{Contexte} : $\sim$4-8K tokens max
\end{itemize}

\section{Contraintes Intégration}

Pas d'intégration ERP native, pas de connecteurs IoT/SCADA, API REST uniquement, authentification Supabase uniquement.
